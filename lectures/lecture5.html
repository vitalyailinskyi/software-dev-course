Big-O notation

Big O notation is used to describe the performance or complexity of an algorithm. Big O specifically describes the worst-case scenario, and can be used to describe the execution time required by an algorithm.
http://rob-bell.net/2009/06/a-beginners-guide-to-big-o-notation/

Common examples

  • O(1): Accessing Array Index (int a = array[5])
  • O(N): Searching for value in array
  • O(N^2): Bubble sort
  • O(log N): Binary search

Collections: interfaces and implementations

  • List - finite ordered collection of values. ArrayList<T>, LinkedList<T>
  • Set - finite collection of unique values. HashSet<T>, TreeSet<T>
  • Map - collection of {key, value} pairs, such that each possible key appears at most once in the collection. HashMap<T>, TreeMap<T>

Example: List

						
List<Integer> list = new ArrayList<Integer>;
list.add(1); list.add(3); list.add(1); // [1, 3, 1]
list.size(); // 3
list.insert(0, 9); // [9, 1, 3, 1]
list.remove(2); // [9, 1, 1] 
list.contains(3); // false

LinkedList<Integer> stack = new LinkedList<Integer>;
stack.push(1); // [1]
stack.push(2); // [2, 1]
stack.push(3); // [3, 2, 1]
stack.peek(); // 3, stack = [3, 2, 1]
stack.pop(); // 3, stack = [2, 1]

Example: Set

						
Set<Integer> set = new HashSet<Integer>;
set.add(1); set.add(3); set.add(1); // { 1, 3 }
set.size(); // 2
set.remove(1); // { 3 }
list.contains(3); // true

Example: Map

						
Map<String, Integer> map = new HashMap<String, Integer>;
map.put("one", 1); // { "one" -> 1 }
map.put("two", 2); // { "one" -> 1, "two" -> 2 }
map.put("three", 3); // { "one" -> 1, "two" -> 2, "three" -> 3 }
map.containsKey("four"); // true
